## L1 · Senior-PM / Startup lens

- **Wedge:** current, personalized, cross-border "enough number" with a stay-or-go
  ranking — genuinely unserved. Keep it sharp; resist becoming a generic FIRE calc.
- **Riskiest assumption:** that the analytical target user *trusts* asking-price-based,
  simplified-tax numbers enough to act and share. If trust fails, nothing else matters.
  → invest in transparency + fidelity before features.
- **Growth loop:** the shareable URL is the engine — but a shared link currently has no
  rich preview. A per-scenario **Open Graph card** ("My enough number: Hyderabad,
  $1.63M — 61% funded") would make every share a billboard. High-leverage, do-able.
- **Retention:** a calculator is inherently low-retention. Don't fight it — optimize for
  *one great share*, not return visits. (A "your number moved as markets shifted" hook is
  a v-next idea, not now.)
- **Measurement gap:** we ship blind. Add privacy-friendly analytics (e.g. Vercel/Plausible)
  to see completion, shares, which controls get touched. Define success metrics first.
- **Headline takeaway:** the next version should be **"Trust & Spread"** — make the
  numbers auditable, and make the share worth spreading.
